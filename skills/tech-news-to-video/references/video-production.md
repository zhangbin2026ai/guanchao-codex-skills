# Video Production

Default format: vertical 9:16, natural Chinese voiceover, burned-in Chinese subtitles, fast but intelligible pacing, about 15 minutes. The video should feel like a news investigation plus commercial-war film plus chessboard/game analysis.

## Asset Package

When a video tool is available, prepare:

- Main voiceover script split into scenes.
- SRT subtitle text with short lines.
- Scene-by-scene visual prompts.
- Topic-relevant dynamic image/video material plan. Match the visual subject to the narration instead of using generic backgrounds.
- Visual evidence for the power shift: product, factory, city, map, financial chart, supply chain, interface, vehicle, device, data center, or human behavior.
- On-screen keyword cards.
- B-roll suggestions.
- Cover text and thumbnail concept.

When no video tool is available, still render a downloadable MP4 with local tools if possible. Only provide assets without a video file when rendering is technically blocked.

## Scene Rhythm

- 0:00-0:10: hook, contradiction, stakes.
- 0:10-1:00: explain the event and why it matters now.
- 1:00-4:00: actors and board.
- 4:00-8:00: hidden constraints and industry chain.
- 8:00-12:00: geopolitics/business-war layer when supported.
- 12:00-14:30: ordinary-person translation.
- 14:30-15:00: contrarian conclusion and comment prompt.

## Visual Prompt Style

Use realistic editorial visuals, technology infrastructure, newsroom graphics, maps, chip fabs, server rooms, power grids, supply-chain diagrams, product closeups, automotive factories, roads, dashboards, launch stages, city streets, financial charts, platform interfaces, and abstract board-game metaphors. Avoid cyberpunk as the default. Avoid using real person likenesses unless the user provides permission or the image is clearly a public-domain/news-safe reference.

Each visual should answer one of these:

- What entrance is being captured?
- Which power is moving?
- Who is becoming dependent?
- What asset is being repriced?
- What does the破局 path look like?

## Subtitle Rules

- Keep each subtitle line short enough for vertical mobile.
- Position subtitles above the bottom platform UI safe zone. Use roughly bottom 18-24% of the frame, not the extreme bottom edge.
- Use punctuation to preserve spoken rhythm.
- Highlight key phrases as optional on-screen text.
- Avoid dense paragraphs on screen.

## Narration Rules

- Do not clone a real creator's voice or train on scraped voice samples without explicit permission.
- Prefer a mature male Chinese narrator with calm sharpness, story pressure, and natural pauses.
- Avoid robotic Windows system voices for final delivery when a better local, plugin, or online TTS option is available.
