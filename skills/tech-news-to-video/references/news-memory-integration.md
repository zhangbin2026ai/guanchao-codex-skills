# News Memory And Integration

Use this reference before selecting or expanding any “观潮” topic. The goal is to avoid writing one isolated news item to the end. Build a memory graph across recent news, older comparable cases, geopolitical constraints, financial pricing, and ordinary-user consequences.

## Core Principle

Do not simply add a keyword such as “security,” “geopolitics,” or “finance.”

Understand the user's underlying logic:

- one small news item may be the visible opening of a larger rule change
- the real story often appears only after connecting multiple news items
- the best script moves from one event to a broader pattern, then returns to the original event with sharper meaning
- the topic should feel like a board being revealed, not a theme being stretched

## News Memory Graph

For every candidate topic, build a compact graph before writing.

### 1. Anchor News

The fresh event that gives the script timeliness.

Ask:

- What happened today or recently?
- Why does it look ordinary on the surface?
- What question does it quietly open?

### 2. Adjacent News

Find 2-5 related events from recent months or years.

Look for:

- similar regulatory logic
- similar supply-chain tension
- similar security concern
- similar capital repricing
- similar company behavior
- similar country-system conflict
- a reversed example from another country or industry

### 3. Common Hidden Rule

Do not list related news mechanically. Extract the shared rule.

Examples:

- enterprise AI partnerships + Tesla FSD China + Huawei export difficulty may share one hidden rule: software is becoming infrastructure, so it is managed as security.
- smart glasses + maps + advertising + payments may share one hidden rule: the first visual layer can become a toll gate.
- AI data centers + energy grid + chip export controls may share one hidden rule: compute power is becoming a strategic resource, not just a business cost.

### 4. Power Layers

Classify the graph through these layers:

- user layer: what ordinary people, workers, consumers, or creators will feel
- company layer: who gets customers, workflows, data, distribution, or tolls
- industry layer: which supply chain, standard, or capability is being rearranged
- national security layer: which data, map, compute, chip, cloud, car, or model becomes sensitive
- geopolitical layer: which country-system distrust appears
- financial layer: which asset, stock story, capex path, or future rent is being repriced

### 5. Return Path

After expanding, return to the anchor news.

The final meaning should be:

> This small event matters because it is one visible move in a larger pattern.

If the script cannot return to the anchor news, the expansion is decorative and should be cut.

## Expansion Ladder

Use this rising ladder when a topic is complex:

1. Start with one concrete event.
2. Explain the ordinary surface meaning.
3. Reveal the practical obstacle.
4. Bring in one comparable case.
5. Extract the shared rule.
6. Add geopolitical or financial layer only if it emerges from the shared rule.
7. Return to what this means for the original event.
8. Translate to ordinary people, companies, or workers.

Do not jump directly from one event to a grand conclusion.

## Memory Prompts

Before drafting, answer:

- What previous news does this remind me of?
- Is there a reverse case from another country?
- Is one side allowing entry with local control while another side blocks entry with distrust?
- Is this about product quality, or about who is allowed to touch data, roads, money, hospitals, factories, maps, or compute?
- Is capital already pricing the hidden rule?
- Is the visible company only a tool for a deeper system conflict?
- What would a normal viewer understand first, and what would they need explained before the bigger pattern makes sense?

## Writing Rule

The integrated script should feel like thinking, not cataloging.

Avoid:

- “还有，还有，还有” lists of related news
- calling every topic “security” without explaining the object being protected
- forcing geopolitics into product news
- forcing finance into every paragraph
- staying with the first theme after a better deeper theme appears

Prefer:

- one anchor scene
- one comparison that makes the hidden rule visible
- one financial or geopolitical consequence that follows naturally
- one return sentence that makes the anchor news feel newly important

## Today’s User-Approved Lesson

The OpenAI / Data Section example should not stop at “enterprise AI improves efficiency.”

The user saw a deeper path:

1. OpenAI needs a local cloud partner because enterprise AI touches data, accountability, and local trust.
2. This connects to Tesla FSD entering China, where software touches roads, maps, camera data, and real-world safety.
3. It also connects to Huawei and Chinese software / compute going abroad, where foreign systems may treat Chinese technology as a security risk.
4. The common hidden rule is not “security” as a slogan. The rule is that software has become a border-crossing infrastructure layer.
5. The story should then ask who is allowed to run software, read data, map roads, rent compute, or enter enterprise workflows inside another system.

Use this as a model for future integration: one news item opens the door, but the story is found by connecting the door to other doors.
